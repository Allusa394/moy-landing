## Часть 2. Структура идеального лендинга

Структура построена по модели AIDA (Attention → Interest → Desire → Action):

### Блок 1. Hero-секция (Первый экран)

**Цель:** Захватить внимание за 3-5 секунд.

**Содержание:**
- Заголовок-оффер (не "Привет, я Вася", а "Делаю сайты, которые приносят клиентов")
- Подзаголовок — конкретика: для кого, какой результат
- CTA-кнопка ("Обсудить проект", "Посмотреть работы")
- Визуальный элемент: анимация, мокап лучшей работы

**Почему важно:** 57% времени пользователь проводит на первом экране. Нет ясного оффера — уходит.

---

### Блок 2. Социальное доказательство (мини-блок)

**Цель:** Быстро снять первичное недоверие.

**Содержание:**
- Логотипы клиентов (если есть)
- Цифры: "50+ проектов", "7 лет опыта", "клиенты из 12 стран"
- Или строка: "Работал с: Сбер, Яндекс, стартапы из YC"

**Почему важно:** Социальное доказательство на раннем этапе снижает «барьер скепсиса».

---

### Блок 3. Услуги / Что я делаю

**Цель:** Показать спектр компетенций.

**Содержание:**
- 3-5 карточек с иконками
- Каждая: название, описание (1-2 предложения), результат для клиента
- Пример: "Лендинги под ключ", "Веб-приложения", "AI-агенты"

**Почему важно:** Посетитель должен найти свою задачу в списке, иначе подумает "это не для меня".

---

### Блок 4. Портфолио / Кейсы

**Цель:** Доказать экспертизу реальными работами.

**Содержание:**
- 4-6 лучших проектов (не все подряд!)
- Каждый: скриншот/мокап, название, задача и результат
- Кликабельные карточки

**Почему важно:** 73% клиентов принимают решение на основе портфолио. Показывать результат, а не процесс.

---

### Блок 5. Процесс работы

**Цель:** Снять тревогу "а что будет после оплаты?"

**Содержание:**
- 4-6 шагов с номерами/таймлайн
- Пример: Бриф → Прототип → Дизайн → Вёрстка → Тестирование → Запуск
- Краткое пояснение к каждому шагу

**Почему важно:** Прозрачный процесс снижает риск. Клиент видит систему, а не хаос.

---

### Блок 6. Отзывы

**Цель:** Голос реальных клиентов — финальный удар по сомнениям.

**Содержание:**
- 3-5 отзывов с именем, должностью, фото
- Конкретика: "сделал лендинг за 5 дней, конверсия выросла на 40%"
- Ссылки на профили авторов

**Почему важно:** 88% потребителей доверяют онлайн-отзывам как личным рекомендациям. Конкретные отзывы с цифрами конвертируют в 2.3x лучше общих.

---

### Блок 7. Обо мне

**Цель:** Установить человеческую связь.

**Содержание:**
- Фото (живое, не стоковое)
- 2-3 абзаца: кто ты, почему этим занимаешься, чем отличаешься
- Ключевые факты: опыт, стек, подход
- Личная деталь (хобби, ценности)

**Почему важно:** Люди покупают у людей. Блок НЕ первый — сначала ценность, потом знакомство.

---

### Блок 8. FAQ

**Цель:** Снять оставшиеся возражения.

**Содержание:**
- 5-7 вопросов в формате аккордеона
- "Сколько стоит?", "Какие сроки?", "Работаете по договору?", "Какая предоплата?"

**Почему важно:** FAQ закрывает возражения, которые клиент стесняется задать. Наличие FAQ увеличивает конверсию на 15-25%.

---

### Блок 9. Контакты / Финальный CTA

**Цель:** Максимально простой способ начать сотрудничество.

**Содержание:**
- Заголовок-призыв: "Давайте обсудим ваш проект" (не сухое "Контакты")
- Форма обратной связи (максимум 3-4 поля)
- Альтернативные способы: Telegram, WhatsApp, email

**Почему важно:** Каждое лишнее поле снижает конверсию на 7-10%.

---

### Путь клиента по секциям

| Этап AIDA | Блоки | Вопрос клиента |
|-----------|-------|----------------|
| **Attention** | 1. Hero | "Что тут для меня?" |
| **Interest** | 2. Social Proof, 3. Услуги | "Решит ли он мою задачу?" |
| **Desire** | 4. Портфолио, 5. Процесс, 6. Отзывы | "Он реально это умеет?" |
| **Action** | 7. Обо мне, 8. FAQ, 9. CTA | "Как начать?" |

---

### Типичные ошибки

1. **Hero без конкретного оффера** — "Привет, я веб-разработчик" это не оффер
2. **Портфолио-свалка** — 5 лучших работ > 20 посредственных
3. **Нет промежуточных CTA** — кнопки "Обсудить проект" нужны после портфолио и отзывов
4. **Стена текста** — карточки, иконки, буллеты сканируются за секунды
5. **Нет мобильной адаптации** — 60%+ трафика с телефонов
6. **Цены спрятаны** — хотя бы "от X ₽"
7. **Форма с 10 полями** — максимум: имя, способ связи, пара слов о задаче
8. **Блок "Обо мне" первым** — сначала ценность, потом знакомство

## Часть 3. Тренды веб-дизайна 2025-2026

### 1. Scroll-Driven Animations (CSS)

Новый CSS API `scroll-timeline` и `view-timeline` — анимации при скролле без JavaScript.

```css
.card {
  animation: fade-in linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 100%;
}
```

Fallback через `IntersectionObserver` в 10 строках JS. Практически все топовые сайты Awwwards 2025 используют.

---

### 2. Bento Grid Layout

Асимметричная сетка карточек разного размера (как виджеты iOS). Идеально для секций «Услуги» и «Проекты».

```css
.bento-grid {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-rows: 200px;
  gap: 16px;
}
.wide { grid-column: span 2; }
.tall { grid-row: span 2; }
.big  { grid-column: span 2; grid-row: span 2; }
```

---

### 3. Glassmorphism 2.0

Полупрозрачные карточки с blur и градиентным свечением по краям. Идеально на тёмном фоне с оранжевым акцентом.

```css
.glass-card {
  background: rgba(255, 255, 255, 0.05);
  backdrop-filter: blur(20px);
  border: 1px solid rgba(255, 255, 255, 0.08);
  border-radius: 20px;
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.4);
}
```

---

### 4. Aurora Backgrounds (градиентные фоны)

Многоцветные градиенты, имитирующие северное сияние. Анимированные градиенты создают ощущение «живого» сайта.

```css
.aurora-bg {
  background:
    radial-gradient(ellipse at 20% 50%, rgba(255, 140, 50, 0.15) 0%, transparent 50%),
    radial-gradient(ellipse at 80% 20%, rgba(255, 100, 0, 0.1) 0%, transparent 50%),
    #0a0a0a;
  animation: aurora-shift 15s ease-in-out infinite alternate;
}
```

---

### 5. Микро-взаимодействия (Micro-interactions)

Кнопки «пульсируют», карточки приподнимаются, иконки анимируются. Ощущение живого интерфейса.

```css
.btn:hover {
  transform: translateY(-3px) scale(1.02);
  box-shadow: 0 10px 40px rgba(255, 140, 0, 0.3);
}
/* Магнитное свечение */
.btn::after {
  background: linear-gradient(135deg, #ff8c00, #ff4500);
  filter: blur(12px);
  opacity: 0;
  transition: opacity 0.3s;
}
.btn:hover::after { opacity: 1; }
```

---

### 6. Dark Mode Glow (тёмная тема со свечением)

Не просто тёмный фон, а «свечение» элементов. Рекомендуемая палитра:

```css
:root {
  --bg-primary: #0a0a0b;
  --bg-secondary: #111113;
  --bg-card: #16161a;
  --text-primary: #e8e8ed;
  --text-secondary: #a0a0b0;
  --accent: #ff8c00;
  --accent-glow: rgba(255, 140, 0, 0.15);
  --border: rgba(255, 255, 255, 0.06);
}
```

---

### 7. Кинетическая типографика

Крупные заголовки с негативным трекингом, градиентной заливкой, анимацией по буквам.

```css
.hero-title {
  font-size: clamp(2.5rem, 8vw, 6rem);
  font-weight: 800;
  letter-spacing: -0.03em;
  background: linear-gradient(135deg, #fff, #ff8c00);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}
```

---

### 8. Tilt-эффекты и параллакс на курсоре

Карточки наклоняются вслед за курсором — премиальное ощущение.

```javascript
card.addEventListener('mousemove', e => {
  const x = (e.clientX - rect.left) / rect.width - 0.5;
  const y = (e.clientY - rect.top) / rect.height - 0.5;
  card.style.transform = `perspective(800px) rotateY(${x*10}deg) rotateX(${-y*10}deg)`;
});
```

---

### 9. AI-тематический дизайн

Для создателя AI-агентов: нейросетевые графы, пульсирующие «нейроны», typewriter-эффект, фоновые сетки данных.

```css
/* Сетка данных в стиле терминала */
.data-grid-bg {
  background-image:
    linear-gradient(rgba(255, 140, 0, 0.03) 1px, transparent 1px),
    linear-gradient(90deg, rgba(255, 140, 0, 0.03) 1px, transparent 1px);
  background-size: 40px 40px;
}

/* Пульсирующие нейроны (SVG) */
.neuron { animation: pulse 2s ease-in-out infinite; }
.synapse { stroke-dasharray: 5 10; animation: data-flow 3s linear infinite; }
```

---

### 10. Горизонтальный скролл секций

Секция с проектами скроллится горизонтально при вертикальной прокрутке. Выглядит эффектно.

```css
.horizontal-scroll {
  overflow-x: auto;
  scroll-snap-type: x mandatory;
  display: flex;
  gap: 24px;
}
.project-card {
  min-width: 320px;
  flex-shrink: 0;
  scroll-snap-align: start;
}
```

---

### 11. Текстовые маски и Clip-Path

Анимированный градиент сквозь текст. Нестандартные формы секций.

```css
.masked-heading {
  background: linear-gradient(135deg, #ff8c00, #ff4500, #fff, #ff8c00);
  background-size: 200% 200%;
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  animation: gradient-move 5s ease infinite;
}
```

---

### 12. Зернистые градиенты (Grain/Noise)

Лёгкий шум поверх градиентов для ощущения глубины и «аналоговости». Тренд 2025 — антипод гладкости.

Реализуется через SVG-фильтр `feTurbulence` в pseudo-элементе с `opacity: 0.04`.

## Часть 4. Копирайтинг и позиционирование

### Примеры заголовков для hero-секции

1. **«Превращаю идеи в сайты, приложения и AI-решения, которые работают на ваш бизнес»**
2. **«Ваш цифровой партнёр: от лендинга до AI-агента за недели, а не месяцы»**
3. **«Создаю цифровые продукты, которые приносят клиентов, пока вы спите»**
4. **«Сайты. Приложения. AI-агенты. Один разработчик — вместо целой команды»**
5. **«Не просто сайт — цифровая система, которая продаёт, отвечает клиентам и масштабируется»**
6. **«От идеи до запуска: веб, мобайл, AI — всё в одних руках»**
7. **«Разрабатываю технологии, чтобы ваш бизнес рос быстрее»**

**Что работает:** конкретика > абстракции, язык клиента > жаргон разработчика, результат > процесс.

---

### Описание услуг: продавать результат, не технологию

**Плохо:** «Разработка веб-приложений на React с серверным рендерингом»
**Хорошо:** «Быстрые и удобные веб-приложения, которые нравятся пользователям и приносят заявки»

**Сайты:**
> Ваш сайт — это первое впечатление. Если он тормозит, выглядит устаревшим или неудобен на телефоне — клиенты уходят к конкурентам. Я создаю быстрые, современные сайты, с которых приходят заявки.

**Приложения:**
> Когда бизнес перерастает Excel-таблицы и чаты в мессенджерах, нужно приложение. Не раздутая ERP за миллионы, а аккуратный инструмент под ваши задачи.

**AI-агенты:**
> AI-агент — это сотрудник, который работает 24/7, не болеет и не просит повышения. Отвечает на вопросы клиентов, обрабатывает заявки, анализирует данные. Стоит дешевле одного месяца зарплаты менеджера.

### Блок «Обо мне» — структура

1. Кто ты и чем занимаешься (1 предложение)
2. Почему тебе можно доверять (опыт, цифры)
3. Что тебя отличает (ценности, подход)
4. Человеческая деталь (хобби, факт)

**Пример:**
> Привет! Меня зовут [Имя], я разработчик с [X] годами опыта. За это время сделал [N] проектов — от одностраничников для кофеен до AI-ассистентов для онлайн-школ. Работаю один, но это плюс: вы общаетесь напрямую с тем, кто делает. Без менеджеров, без испорченного телефона.

### CTA — лучшие формулировки

**Мягкие:** «Обсудить проект», «Получить бесплатную консультацию», «Рассказать о задаче»
**Средние:** «Получить оценку проекта», «Узнать стоимость»
**Активные:** «Запустить проект», «Начать работу»

**Стратегия размещения:**
1. Hero — главный CTA ("Обсудить проект")
2. После портфолио — "Хотите так же?"
3. После тарифов — "Выбрать тариф"
4. Футер — финальный CTA + контакты

**Подпись под кнопкой:** «Бесплатно, без обязательств» или «Отвечу в течение часа»

### Тарифы — как не отпугнуть

**Рекомендуемый подход — 3 пакета:** Базовый → Стандарт (выделен как «Популярный») → Премиум

**Психологические приёмы:**
- Цена за день/месяц: «AI-агент — 1500₽/мес вместо менеджера за 50 000₽/мес»

- Гарантия: «Если результат не устроит — верну предоплату»

## Часть 5. CSS/JS эффекты без библиотек

### Hero-секция: 5 эффектов

**1. Typing Animation (чистый CSS)**
```css
.typing {
  width: 0;
  overflow: hidden;
  white-space: nowrap;
  border-right: 3px solid #ff6a00;
  animation: typing 3s steps(20) forwards, blink 0.7s step-end infinite;
}
@keyframes typing { to { width: 20ch; } }
@keyframes blink { 50% { border-color: transparent; } }
```

**2. Градиентный анимированный текст**
```css
.gradient-text {
  background: linear-gradient(270deg, #ff6a00, #ff9a44, #fff, #ff6a00);
  background-size: 300% 300%;
  -webkit-background-clip: text;
  color: transparent;
  animation: gradient-shift 6s ease infinite;
}
```

**3. Particle Background (Canvas, vanilla JS)**
Canvas на весь экран, массив частиц с координатами/скоростью, линии между близкими частицами через `ctx.lineTo()`. Цвета — оранжевый с разной прозрачностью. 50-80 частиц на десктопе, 20-30 на мобильных.

**4. Glow Pulsation**
```css
.glow {
  color: #ff6a00;
  animation: glow-pulse 2s ease-in-out infinite alternate;
}
@keyframes glow-pulse {
  from { text-shadow: 0 0 5px #ff6a00, 0 0 10px #ff6a00, 0 0 20px #ff4500; }
  to   { text-shadow: 0 0 10px #ff6a00, 0 0 25px #ff6a00, 0 0 50px #ff4500; }
}
```

**5. SVG Morphing фон**
```html
<svg><path fill="rgba(255,106,0,0.1)">
  <animate attributeName="d" dur="8s" repeatCount="indefinite"
    values="M0,250 C150,100 350,400 500,250...;M0,250 C150,400 350,100 500,250..." />
</path></svg>
```

### Scroll-анимации: 5 техник (IntersectionObserver + CSS)

**Общий JS (10 строк на все анимации):**
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('visible');
      observer.unobserve(entry.target);
    }
  });
}, { threshold: 0.15 });
document.querySelectorAll('.animate-on-scroll').forEach(el => observer.observe(el));
```

**1. Fade-up:**
```css
.fade-up { opacity: 0; transform: translateY(40px); transition: all 0.6s ease; }
.fade-up.visible { opacity: 1; transform: translateY(0); }
```

**2. Каскадное появление (Stagger):**
```css
.stagger-item { opacity: 0; transform: translateY(30px); transition: all 0.5s ease;
  transition-delay: calc(var(--i) * 0.1s); }
.stagger-item.visible { opacity: 1; transform: translateY(0); }
```

**3. Slide-in (горизонтальное выезжание):**
```css
.slide-left { opacity: 0; transform: translateX(-60px); transition: all 0.7s ease; }
.slide-left.visible { opacity: 1; transform: translateX(0); }
```

**4. Scale-reveal:**
```css
.scale-reveal { opacity: 0; transform: scale(0.85); transition: all 0.6s ease; }
.scale-reveal.visible { opacity: 1; transform: scale(1); }
```

**5. Прогресс-бар навыков:**
Полоска с `width: 0` → `width: var(--skill-level)` при `visible`. Инлайн: `style="--skill-level: 85%"`.

---

### Hover-эффекты карточек: 3 техники

**1. Gradient border** — `conic-gradient` обводка с вращением при hover.

**2. Tilt-эффект** — наклон карточки за курсором через `mousemove` + `perspective(800px) rotateX/Y`.

**3. Clip-path reveal** — оверлей с `clip-path: circle(0%)` → `circle(75%)` при hover. Красивое «раскрытие» из центра.

---

### Cursor-эффекты: 3 техники

**1. Кастомный курсор с хвостом** — два div с `position: fixed`, основной двигается мгновенно, следящий — через lerp (`followerX += (mouseX - followerX) * 0.08`).

**2. Свечение за курсором** — div с `radial-gradient(rgba(255,106,0,0.15), transparent)` следует за мышью.

**3. Параллакс-элементы** — декоративные SVG-фигуры смещаются при `mousemove` пропорционально «глубине» (`data-depth`).

На мобильных (`matchMedia('(hover: none)')`) — отключать.

### Тёмная тема: best practices

| Элемент | Значение |
|---------|----------|
| Основной фон | `#0a0a0a` — `#121212` (не чисто чёрный!) |
| Карточки | `#1a1a2e` или `#1e1e1e` |
| Основной текст | `#e0e0e0` — `#f0f0f0` (не чисто белый!) |
| Вторичный текст | `#a0a0a0` — `#b0b0b0` |
| Оранжевый акцент | `#ff6a00` / `#ff8c00` |
| Тонкие линии | `rgba(255, 106, 0, 0.3)` |
| Фоновые свечения | `radial-gradient(rgba(255,106,0,0.08), transparent)` |
| Тени | `0 0 20px rgba(255,106,0,0.1)` вместо классических |

**Контраст WCAG AA:** минимум 4.5:1 для текста, 3:1 для крупного.

---

### Производительность на мобильных

1. **Анимировать только `transform` и `opacity`** — они на GPU, не вызывают reflow
2. **`will-change: transform, opacity`** — на анимируемых элементах
3. **`prefers-reduced-motion: reduce`** — обязательно уважать
4. **Canvas:** снижать частицы на мобильных, аккуратно с `devicePixelRatio`
5. **IntersectionObserver вместо scroll-события** — не блокирует главный поток
6. **`content-visibility: auto`** — пропускает рендеринг элементов за экраном

### Продвинутые CSS-техники

**backdrop-filter** — «матовое стекло» для навигации:
```css
.glass-nav {
  background: rgba(10, 10, 10, 0.6);
  backdrop-filter: blur(12px) saturate(180%);
  border-bottom: 1px solid rgba(255, 255, 255, 0.08);
}
```

**clip-path** — нестандартные формы секций, маскирование изображений.

**mix-blend-mode: difference** — текст автоматически контрастирует с любым фоном.
